## 〇、核心要求

本项目**主题自选、自己出题**，但硬性要求统一、不可商量：

1. **题材你自己定**——可以从第一节给出的推荐方向里挑一个，也可以完全自拟，只要落在「**带登录的数据型 App**」这个大框架内即可。
2. **硬性要求**——页面数量、自定义组件数量、代码规模、**本地数据库 ≥ 300 条**、图片 ≥ 30 张、**axios 接口 ≥ 2 个**、数据可视化图表。
3. **两块压轴必做**——**必须用本地关系型数据库 `relationalStore` 承载 ≥300 条结构化数据**，并用 `@ohos/axios` 对接后端。这两块是本项目**重点必考项**。
4. **数据可视化硬指标**——「我的 / 统计」页**必须有 ≥1 个基于数据库聚合结果的可视化图表**。

## 二、项目总体结构（通用骨架 · 按你的主题改）

> 下面是这类「带本地数据库 + 网络的数据型 App」的**典型骨架**。**页面怎么分、Tab 有几个、功能叫什么名,你都按自己主题定**——重要的不是"长得和下图一样",而是**每一层的能力都具备**(本地数据库 / 网络 / 状态管理 / 数据可视化)。

```
启动页（品牌 + 动画）
   │
   ├─ 登录页（拿到 token 并落盘）
   │
   └─ 主页（底部 Tab 容器，几个 Tab 自定）
        ├─ 核心内容页   ← 从本地 RDB 读 ≥300 条，可筛选 / 搜索
        │     └─ 详情页（可收藏）
        │     └─ 一个"有交互深度"的功能页（测验 / 打卡 / 计划 / 记录…任选）
        ├─ 网络功能页   ← axios 拉取 + 提交（点赞 / 评论 / 发布 / 同步…任选）
        └─ 个人中心     ← 登录态切换 + 数据可视化统计
              └─ 收藏页 / 历史页（读 RDB）
```

数据流（以"核心内容 → 收藏 → 统计"为例，换成你主题的实体即可）：

```
RDB(主内容表, 300+条) --query--> 列表页 --pushUrl(id)--> 详情页
                                                        │ 点"收藏"
                                                        ▼
                                            RDB(收藏表) insert/delete
                                                        │
                    个人中心 <--query+聚合(Map)-- 收藏数 / 行为次数 / 分类分布
                            └── 渲染成 ≥1 个可视化图表(Grid / Canvas / 统计图)
```

## 三、功能需求

### 模块 1 · 启动页
- 1.1 启动展示品牌页(Logo + 标语),**带入场动画**(淡入 / 位移等),短暂停留后自动跳主页。【考点：animateTo / transition 动画；router.replaceUrl】
- 1.2 读取本地登录态(PersistentStorage 的 token)决定进登录页还是主页。【考点：PersistentStorage】

### 模块 2 · 登录
- 2.1 用户登录(**方式自选**:手机号+验证码 / 账号+密码 / 其它),做必要的表单校验。【考点：TextInput / Button；表单校验；(若做倒计时) setTimeout / 状态机】
- 2.2 调后端登录接口拿 token,存入 PersistentStorage / AppStorage。【考点：axios POST；AppStorage / PersistentStorage】
- 2.3 登录成功 `router.replaceUrl` 进主页(销毁登录页,返回键不回登录页)。【考点：路由 replaceUrl】

### 模块 3 · 主页框架(底部 Tab)
- 3.1 底部 Tab 导航(**几个 Tab 自定**),切换内容区,选中 / 未选中态切换。【考点：Tabs + TabContent + TabsController】
- 3.2 用 AppStorage 记录当前 Tab 索引,可跨页定位回指定 Tab。【考点：@StorageLink】

### 模块 4 · 核心内容库列表页（**RDB 数据页 · 重点 🔴**）
- 4.1 从**本地 RDB** 读取**你主题的核心内容**列表(主标题 / 副标题 / 分类等)。(例:诗词→标题/作者/朝代;菜谱→菜名/菜系/难度)【考点：relationalStore query + ResultSet 游标】
- 4.2 **按分类筛选**,用谓词条件查询。【考点：RdbPredicates equalTo；enum 分类】
- 4.3 **关键词搜索**(模糊匹配)。【考点：RdbPredicates like；TextInput.onChange】
- 4.4 列表用 **List + ListItem 懒加载**(≥300 条数据**禁止** Column+ForEach 全量渲染)。【考点：List 懒加载】
- 4.5 列表项封装成**自定义组件**。【考点：@Component 自定义组件 + @Prop 传参】

### 模块 5 · 详情页 + 收藏
- 5.1 `router.pushUrl` 传 id,详情页 `getParams` 接收并从 RDB 查该条详情。【考点：路由传参 interface 类型化接收；RDB 单条查询】
- 5.2 **收藏 / 取消收藏**:写入 / 删除 RDB 收藏表,按钮状态实时更新。【考点：RDB insert / delete；@State 响应式】
- 5.3 详情页配背景图 + 内容出现转场。【考点：backgroundImage；transition】

### 模块 6 · 一个"有交互深度"的功能（**自选,不限于测验**）
> 做**一个能体现 计时 / 状态机 / 结果弹窗 / 写库** 的进阶功能即可——**测验 / 打卡 / 计划 / 记录 / 计算器 / 小游戏…任选一个**贴合你主题的。下面以"测验"举例,可整体换成等价功能:
- 6.1 从 RDB 取数据驱动该功能(如测验组卷、打卡生成今日任务、计划排期)。【考点：RDB 查询(如组卷再加数组洗牌)】
- 6.2 **TextTimer 计时** + Progress 进度 + 状态机(如 对/错/未答、已打卡/未打卡)。【考点：TextTimer；enum 状态机】
- 6.3 完成后弹**自定义结果弹窗**(成绩 / 总结 / 再来一次)。【考点：@CustomDialog】
- 6.4 结果写入 RDB 行为 / 历史表。【考点：RDB insert】

### 模块 7 · 一个"走网络"的功能（**自选,不限于社交圈**）
> 做**一个用 axios 既拉取又提交** 的功能即可——**社交圈 / 评论 / 排行榜 / 数据同步 / 内容推荐…任选**。下面以"打卡圈"举例:
- 7.1 调后端接口(GET)**加载列表**展示。【考点：axios GET 请求】
- 7.2 **点赞 / 互动**,列表项用 @Observed + @ObjectLink 字段级深度响应。【考点：@Observed + @ObjectLink；axios POST/GET】
- 7.3 **提交内容**(发布 / 评论 / 上传),提交后刷新列表。【考点：axios POST；刷新通知】
- 7.4 请求统一走 `axios.create` 实例;需要登录的接口在请求头手动带上 token。

### 模块 8 · 个人中心 + 数据可视化
- 8.1 已登录 / 未登录两态;显示用户信息(调后端 GET)。【考点：条件渲染；axios GET + 请求头带 token】
- 8.2 **数据可视化统计(硬指标)**:从 RDB 聚合(如各分类占比 / 收藏数 / 行为次数),用 **Grid 矩阵图 / 自绘 Canvas / 统计图**(三选一,至少 1 个)展示。【考点：RDB 聚合 + Map；Grid 网格 / Canvas 自绘可视化】
- 8.3 跳转**收藏页**和**历史 / 行为记录页**(读 RDB)。【考点：RDB 多表查询；路由】
- 8.4 退出登录(清 token)。【考点：PersistentStorage】

## 四、数据与资源硬性要求（**对所有主题一视同仁，缺一扣分**）

| # | 项 | 要求 |
|---|---|---|
| 1 | **本地数据库** | 必须用 `relationalStore`，**核心内容表 ≥ 300 条**真实/拟真种子数据 |
| 2 | 数据库表 | **≥ 3 张表**：`content`（核心内容库）/ `favorite`（收藏）/ `record`（行为 / 历史记录） |
| 3 | **图片资源** | `resources/base/media` 下 **≥ 30 张**（SVG 图标为主：Tab 选中/未选中、分类、功能图标 + 少量背景图） |
| 4 | **网络接口** | 对接 Mock 后端 **≥ 2 个接口**（如登录 + 列表查询） |
| 5 | 代码规模 | **7到9 页面 + 3~4 自定义组件 ** |
| 6 | **数据可视化** | 个人中心**至少 1 个**基于 RDB 聚合的图表（Grid / 自绘 Canvas / 统计图三选一） |


> 关于 300 条数据怎么来：可手工整理公开数据、写脚本生成、或用 AI 批量生成后人工校验。**数据真实性不卡死，但必须是 ≥300 条结构合理、字段完整的真数据**，不能用 3 条复制 100 遍。

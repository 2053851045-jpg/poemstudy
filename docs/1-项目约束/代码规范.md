## 一、命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 组件文件 | 大驼峰（PascalCase），以 `Page.ets` 或 `.ets` 结尾 | `PoemListPage.ets`、`PoemListItem.ets` |
| 工具/服务类 | 大驼峰 + 具体职责 | `DatabaseHelper.ets`、`ApiService.ets` |
| 变量/函数 | 小驼峰（camelCase） | `loadMoreData()`、`isFavorite` |
| 常量 | 全大写下划线分隔 | `MAX_PAGE_SIZE`、`DEFAULT_TOKEN_KEY` |
| 枚举 | 大驼峰，成员全大写 | `QuizStatus { NOT_STARTED, ANSWERING, COMPLETED }` |
| 接口/类型 | 以 `I` 或直接大驼峰 | `Poem`、`UserInfo` |

## 三、ArkTS 语法要点

### 3.1 状态管理规范

| 场景 | 使用的装饰器 |
|------|-------------|
| 组件内部可变状态 | `@State` |
| 父传子（只读） | `@Prop` |
| 父传子（双向） | `@Link` |
| 跨层级传递 | `@Provide` / `@Consume` |
| 嵌套对象深度更新 | `@Observed` + `@ObjectLink` |
| 应用全局状态 | `AppStorage` / `@StorageLink` |
| 持久化存储（Token） | `PersistentStorage` |

### 3.2 异步处理

- 所有数据库和网络操作必须使用 `async/await` + `try-catch`
- 禁止使用 `.then().catch()` 回调嵌套

```typescript
// ✅ 正确
try {
  const data = await this.db.query(params);
  // 处理数据
} catch (err) {
  console.error('查询失败', err);
  // 显示 Toast
}

// ❌ 错误
this.db.query(params).then(data => { ... });
```

### 3.3 结果集资源释放

**强制要求**：`ResultSet` 使用完毕后必须调用 `close()`，建议放在 `finally` 中。

```typescript
let resultSet: relationalStore.ResultSet | null = null;
try {
  resultSet = await this.rdbStore.query(predicates);
  while (resultSet.goToNextRow()) {
    // 读取数据
  }
} catch (err) {
  console.error('查询失败', err);
} finally {
  resultSet?.close();  // 必须关闭
}
```

## 四、UI 开发规范

| 规范 | 要求 |
|------|------|
| 列表渲染 | ≥20 条数据必须用 `List` + `LazyForEach`，禁止 `ForEach` 全量渲染 |
| 布局优先级 | `Column`/`Row` > `Flex` > `Stack`（除非需要层叠） |
| 颜色管理 | 主题色定义在 `Constants.ets`，禁止硬编码颜色值 |
| 尺寸单位 | 统一使用 `vp`，字体用 `fp` |

## 五、提交消息规范

```
<type>(<scope>): <subject>

类型（type）：
  - feat: 新功能
  - fix: 修复 bug
  - docs: 文档更新
  - style: 代码格式（不影响运行）
  - refactor: 重构
  - test: 测试相关
  - chore: 构建/工具配置

示例：
  feat(poem-list): 实现分类筛选功能
  fix(db): 修复 ResultSet 未关闭导致的内存泄漏
  docs(constraint): 更新数据模型约束 v1.1
```

## 六、代码审查要点

提交 PR/MR 前必须自查：
1. ✅ 所有 RDB 查询是否关闭了 ResultSet？
2. ✅ 网络请求是否有 try-catch 和 Toast 提示？
3. ✅ 页面销毁时是否释放了资源（如定时器、事件监听）？
4. ✅ 自定义组件是否按文件单独存放？
5. ✅ 是否有未使用的 import 或 console.log？

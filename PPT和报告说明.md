# 2026 移动 APP 应用软件开发 期末考试 · 报告与答辩说明

> **主题自选、1-3 人小组（最多 3 人）完成**

---

## 一、提交材料总览（每组一份）

每组需提交以下材料:

1. **完整的项目代码**
2. **项目技术报告**(电子版,不少于 15 页)
3. **答辩 PPT**(用于现场答辩)

---

## 二、项目技术报告要求

### 2.1 基本要求
1. **页数**:不少于 15 页。
2. **形式**:**1-3 人小组（最多 3 人）**完成并提交
3. **描述对象**:全程围绕**你自己选的主题 App**展开,不要套用别人的题材。

### 2.2 报告内容结构（按"背景 → 需求 → 设计 → 实现 → 过程 → 总结"逐层展开）

**第一部分:项目背景与概述(1-2 页)**
- **选题背景与意义**:为什么选这个题材、想解决什么问题 / 满足什么需求、同类 App 的现状与不足
- **App 简介**:一句话概括核心功能 + 主要特色
- **开发环境与技术栈**:DevEco Studio 版本、ArkTS / Stage 模型、SDK 版本、用到的三方库(@ohos/axios 等)

**第二部分:需求分析(1 页)**
- **功能需求**:列出你的 App 要实现哪些功能
- **用户场景**:典型用户是谁、在什么场景下、要完成什么任务

**第三部分:系统设计与架构(2-3 页)**
- **代码结构与分层**:model / services / components / pages 各层职责
- **功能模块划分**:模块关系(可画结构图)
- **数据库设计(重点)**:≥3 张表(核心内容表 ≥300 条 / 收藏表 / 历史表)的字段、关系、建表语句与设计理由 —— 体现 `relationalStore`
- **状态管理设计**:@State / @Prop / @Link / @Observed 等如何分工
- **页面导航结构**:可画路由图说明
- **关键数据结构**:interface / enum 设计

**第四部分:关键技术实现(3-4 页)** —— 选 **2-3 个**你认为最有挑战性的技术点详细说明
- **本地数据库 relationalStore**:建表、CRUD、`RdbPredicates` 谓词(筛选/模糊/排序)、`ResultSet` 游标遍历、Model 单例封装、≥300 条数据如何组织与聚合统计
- **网络请求 @ohos/axios**:`axios.create` 实例、GET / POST 请求、请求头携带 token、两层拆包(`res.data.result`)
- **状态管理机制**:@State/@Prop/@Link 场景、AppStorage/PersistentStorage、`@Observed`+`@ObjectLink` 列表项深度响应(如点赞)
- **数据可视化**:RDB 聚合 + Grid 矩阵图 / Canvas 自绘图表
- **动画与体验**:转场动画、列表懒加载、按钮状态视觉反馈

**写作要求:** 每个技术点要有**代码片段**说明 + 解释**为什么这样实现** + **遇到的问题和解决方案**。

**第五部分:开发过程与问题解决(2-3 页)**
- 开发时间线(各模块开发顺序、各阶段主要工作、进度安排)
- 遇到的主要问题(技术难点与解决过程、调试思路、从网上/AI/其他渠道获得的帮助)

**第六部分:总结与反思(1 页)**
- 完成情况、技术收获、团队分工与协作、不足与改进方向

---

## 三、答辩 PPT 要求

现场答辩用 PPT,建议包含以下 6 部分:

1. **项目介绍** —— 主题背景、功能介绍、选题理由
2. **系统架构** —— 技术层次、**数据库设计**、状态管理、模块关系
3. **功能演示** —— 核心内容库(**重点**)、登录、你的交互功能、你的网络功能、**数据可视化图表**
4. **技术亮点** ——  relationalStore 数据库、 axios 封装、状态管理、自定义组件、动画与体验优化
5. **开发心得** —— 踩的坑与解决、调试思路、收获
6. **项目总结** —— 完成情况、技术成果、个人成长
